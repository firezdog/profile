Music Listening Site

Previous Next

Group project -- Python capstone for first month of Coding Dojo. The assignment was to produce a Spotify imitation site with functional registration / login, user playlists, and music search. We used Napster API to pull music information and track samples; I was responsible for login and registration, styling (Semantic UI), the index page, and a dynamic search feaature.

Features

  • Login and registration with error messages.
  • User information and liked songs stored in database.
  • Music information and sample playback from Napster API.
  • Track page displaying song information and similar artists.
  • User page displaying all liked songs.
  • Artist page displaying popular tracks.
  • Dynamic search bar groups results by artist as user types.
  • Used Semantic UI for styling.

Multiplayer Tic Tac Toe

Previous Next

Group project -- MEAN capstone for second month of Coding Dojo. The assignment was to create a multiplayer game. Once users login, they are able to view a list of available games, join or create a game, and play with other users. User scores are logged after each section, and a scoreboard keeps track of player rank. Scoreboard can be sorted either by percentage or number of wins. I collaborated with group members on every aspect of the project but was especially involved in login and registration, game logic, and socket troubleshooting.

Features

  • Angular framework and socket technology creates a seamless and interactive experience.
  • Login and registration with dynamically updating error messages.
  • Users can create games or view and join open games.
  • Scoreboard displays player ranking across multiple dimensions.
  • Chat feature during play.

Ninjaman

Introductory challenge for Coding Dojo; uses vanilla JavaScript and HTML. Player moves Ninja through randomly generated maps using the arrow keys, collecting points by eating sushi randomly generated at fixed intervals of time. A pumpkin tracks the player's location and selects a quick route to the player -- if the pumpkin catches the player, she dies. Player has three lives and running score, which is reset upon final death.

Features

  • Player can move ninja across board
  • Randomly generated maps and sushi
  • Three lives and running score
  • Enemy tracks player location

Simon Game

Part of Free Code Camp's Front End Development Certificate; uses Javascript with jQuery, HTML, and CSS. Recreates basic functionality of the popular Simon game: each turn the computer pushes an increasingly long sequence of randomly selected buttons which the player must then reproduce. Each button has its own color, animation, and sound. A strict mode restarts the game upon mistakes; regular mode allows the player to go back and try again.

Features

  • Buttons play sounds and flash animations.
  • Computer and player take turns reproducing a random sequence of button pushes.
  • Strict and regular modes allow player to customize challenge.

Calculator

Part of Free Code Camp's Front End Development Certificate; uses Javascript with jQuery, HTML, and CSS. The calculator performs basic arithmetic operations in the correct order and updates a running sum as new operations are entered. In order to create the calculator, I had to think carefully about different use cases; I also had to find an effective way of representing and allowing the calculator to track the application state. I have detailed my thought process in a blog entry here.

Features

  • Multiplication, division, subtraction, addition, clear entry, clear total
  • Running sum as operations are entered respects order of operation
  • Bootstrap layout mimics real-life calculator
